### 2.2 三大主力工具深度评测

#### 🏆 v0.dev（Vercel 出品）

**定位**：AI驱动的UI组件生成器，专注于前端视觉质量

**核心优势**：
- 生成代码质量最高：输出干净的 React + shadcn/ui + Tailwind CSS
- 对话式迭代：可以反复微调单个组件的细节
- 支持图片输入：上传设计稿截图即可生成对应代码
- 部署无缝：一键部署到 Vercel

**劣势**：
- **仅前端**：无数据库、无认证、无后端API
- 仅限 React/Next.js 生态
- 免费额度有限，$20/月的credits按模型消耗

**定价详情**（2026年最新）：
| 方案 | 价格 | 额度 |
|------|------|------|
| Free | $0 | 有限生成次数 |
| Premium | $20/月 | $20 credits + 优先访问 |
| Premium Pro | $50/月 | 更高额度 |

**适合 MysticMirror 的场景**：生成塔罗牌展示组件、星座运势卡片、五行八卦可视化组件

#### 🏆 Lovable.dev（原 GPT Engineer）

**定位**：面向非技术创始人的全栈应用构建器

**核心优势**：
- 深度集成 Supabase：自动生成认证、数据库schema、API
- 从文字prompt到可部署应用仅需几分钟
- 支持 Figma-to-code 工作流
- GitHub同步：可推送代码到自有仓库
- SOC 2 Type II + ISO 27001 认证（安全性最好）

**劣势**：
- 仅支持 React + Vite（不支持 Next.js）
- 信用点消耗快，重度使用实际月费可达 $50-100+
- 复杂应用的代码可维护性需要后期重构

**定价详情**：
| 方案 | 价格 | 额度 |
|------|------|------|
| Free | $0 | 5 credits/天 |
| Starter | $20/月 | 50 credits/月 |
| Pro | $25-50/月 | 100 credits/月 |

**真实项目成本参考**：
| 项目类型 | Lovable | Bolt.new | v0 |
|----------|---------|----------|-----|
| 简单Landing Page | $0-25 | $0-25 | $0-20 |
| SaaS MVP（含认证+数据库） | $50-100/月 | $50-75/月+数据库 | 不适用 |
| 电商站 | $100-200/月 | $75-150/月 | $20-30/月（仅UI） |

#### 🏆 Bolt.new（StackBlitz 出品）

**定位**：基于 WebContainers 的全栈AI应用原型工具

**核心优势**：
- 浏览器内运行完整 Node.js 环境（WebContainers技术）
- 框架最灵活：支持 React/Next.js/Vue/Astro/Svelte
- V2版本增加内置数据库、认证、存储（Bolt Cloud）
- 免费层最慷慨：1M tokens/月 + 300K/天

**劣势**：
- Token消耗最快（复杂项目3-5天可耗尽10M）
- 代码质量不稳定，大型项目多次迭代后上下文丢失
- 无安全认证（非SOC 2/ISO 27001）

**定价详情**：

## 3. Part 2：AI设计工作流 + 提示词模板

### 3.1 推荐工作流：视觉概念 → 代码生成 → 精修

```
Step 1: 视觉概念生成 (Midjourney V7)
  → 生成Hero背景图、塔罗卡片视觉、配色灵感板
     ↓
Step 2: 页面结构+代码生成 (v0 / Lovable)
  → 文字prompt生成Landing Page结构、组件库
     ↓
Step 3: 精修与定制 (Cursor / GitHub Copilot)
  → 自定义交互逻辑、后端API对接
     ↓
Step 4: 部署与迭代
  → Vercel/Netlify一键部署 + Midjourney持续生成营销素材
```

### 3.2 详细工作流说明

**Phase 1：视觉概念设计（Midjourney）**
1. 在 Midjourney 中生成 3-5 张不同风格的 Hero 背景概念图
2. 生成塔罗牌视觉风格参考（正反面设计）
3. 生成 Logo/Icon 概念（月亮、星辰、八卦、水晶球等元素融合）
4. 使用 `--ar 16:9` 生成宽屏Hero背景，`--style raw` 获得更少MJ风格输出

**Phase 2：代码生成**
- 方案A（最高质量）：v0.dev — 上传 Midjourney 视觉概念图，描述页面结构，迭代微调
- 方案B（最快MVP）：Lovable.dev — 文字描述整个应用，自动生成前端+Supabase后端

**Phase 3：精修与定制（Cursor）**
- 在 Cursor 中打开生成的代码
- 添加自定义交互（塔罗牌翻转动画、八字排盘逻辑等）
- 对接 MysticMirror API

### 3.4 AI建站通用提示词技巧

1. **先定色彩，再写功能**：AI工具对色彩指令非常敏感，先指定具体hex值
2. **指明字体**：不指定的话AI会用默认无衬线字体，不符合灵性主题
3. **分组件生成**：不要一次生成整个网站，先做Hero → 再做Features → 再做Footer
4. **用图片引导**：先在Midjourney生成视觉参考，再上传到v0/Lovable作为风格引导
5. **迭代式优化**：通过对话式迭代逐步精调（"make the gold accents more subtle", "add more whitespace"）
