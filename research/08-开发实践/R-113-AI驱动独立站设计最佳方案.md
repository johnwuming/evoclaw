### 2.2 三大主力工具深度评测

#### 🏆 v0.dev（Vercel 出品）

**定位**：AI驱动的UI组件生成器，专注于前端视觉质量

**核心优势**：
- 生成代码质量最高：输出干净的 React + shadcn/ui + Tailwind CSS
- 对话式迭代：可以反复微调单个组件的细节
- 支持图片输入：上传设计稿截图即可生成对应代码
- 部署无缝：一键部署到 Vercel

**劣势**：
- **仅前端**：无数据库、无认证、无后端API
- 仅限 React/Next.js 生态
- 免费额度有限，$20/月的credits按模型消耗（v0 Max比v0 Mini快5倍消耗）

**定价详情**（2026年最新）：
| 方案 | 价格 | 额度 |
|------|------|------|
| Free | $0 | 有限生成次数 |
| Premium | $20/月 | $20 credits + 优先访问 |
| Premium Pro | $50/月 | 更高额度 |

**适合 MysticMirror 的场景**：生成塔罗牌展示组件、星座运势卡片、五行八卦可视化组件

#### 🏆 Lovable.dev（原 GPT Engineer）

**定位**：面向非技术创始人的全栈应用构建器

**核心优势**：
- 深度集成 Supabase：自动生成认证、数据库schema、API

- 支持 Figma-to-code 工作流
- GitHub同步：可推送代码到自有仓库
- SOC 2 Type II + ISO 27001 认证（安全性最好）

**劣势**：
- 仅支持 React + Vite（不支持 Next.js）
- 信用点消耗快，重度使用实际月费可达 $50-100+
- 复杂应用的代码可维护性需要后期重构

### 3.2 详细工作流说明

#### Phase 1：视觉概念设计（Midjourney）
**目的**：确定整体视觉风格、配色方向、Hero区域视觉

1. 在 Midjourney 中生成 3-5 张不同风格的 Hero 背景概念图
2. 生成塔罗牌视觉风格参考（正反面设计）
3. 生成 Logo/Icon 概念（月亮、星辰、八卦、水晶球等元素融合）
4. 生成配色 mood board

**Midjourney 使用技巧**：
- 使用 `--ar 16:9` 生成宽屏Hero背景
- 使用 `--style raw` 获得更少Midjourney风格的输出
- 使用 `--s 250-400` 提高风格化程度（适合艺术感强的灵性主题）
- 生成后用 Figma 取色器提取配色

#### Phase 2：代码生成（v0 / Lovable）
**目的**：从视觉概念到可运行代码

**方案A（推荐，最高质量）**：v0.dev
1. 上传 Midjourney 生成的视觉概念图到 v0
2. 描述页面结构和功能需求
3. 迭代微调生成的组件
4. 复制代码到自己的 Next.js 项目

**方案B（最快出MVP）**：Lovable.dev
1. 直接用文字描述整个应用
2. Lovable 自动生成前端+后端
3. 连接 Supabase 做数据层
4. GitHub同步获取代码

#### Phase 3：精修与定制（Cursor）
- 在 Cursor 中打开生成的代码
- 添加自定义交互逻辑（如塔罗牌翻转动画）
- 对接 MysticMirror API
- 优化性能和SEO

### 3.4 AI建站通用提示词技巧

1. **先定色彩，再写功能**：AI工具对色彩指令非常敏感，先指定具体hex值
2. **指明字体**：不指定的话AI会用默认无衬线字体，不符合灵性主题
3. **分组件生成**：不要一次生成整个网站，先做Hero → 再做Features → 再做Footer
4. **用图片引导**：先在Midjourney生成视觉参考，再上传到v0/Lovable作为风格引导
5. **迭代式优化**：第一版永远不够好，通过对话式迭代逐步精调（"make the gold accents more subtle", "add more whitespace", "increase the mystical atmosphere"）
