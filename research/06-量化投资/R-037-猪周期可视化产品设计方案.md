## 1. 猪周期核心知识框架

### 1.1 传导链

```
能繁母猪存栏（领先10个月）
  → 仔猪出生/价格（领先6个月）
    → 生猪出栏（供给冲击）
      → 猪肉价格（滞后反应）
        → 养殖利润
          → 补栏/淘汰决策 → 回到能繁母猪
```

这是经典的**蛛网模型（Cobweb Model）**：生产者根据当期价格决定下期供给，但生猪有生物学滞后期，导致价格周期性波动。中国猪周期通常 **3-4 年一轮**。

### 1.2 指标分层

| 层级 | 指标 | 领先/滞后 | 更新频率 | 数据源 |
|------|------|----------|---------|--------|
| **核心领先** | 能繁母猪存栏 | 领先猪价 ~10个月 | 月度 | AKShare `futures_hog_core` / 国家统计局 |
| **核心领先** | 仔猪价格 | 领先 ~6个月 | 周度 | AKShare 现货生猪大数据 |
| **核心同步** | 生猪价格（外三元） | 同步 | 日度 | AKShare `futures_hog_info`（源：养猪数据中心） |
| **核心同步** | 猪粮比/养殖利润 | 同步 | 周度 | AKShare / 公式计算 |
| **辅助同步** | 出栏体重 | 同步，反映压栏/抛售 | 周度 | 涌益咨询（付费）/ 间接推算 |
| **辅助同步** | 屠宰量/开工率 | 需求端 | 周度 | 卓创资讯（付费） |
| **辅助领先** | 二元母猪价格 | 领先 ~8个月 | 周度 | AKShare 现货生猪大数据 |
| **宏观** | 饲料价格（玉米/豆粕） | 成本端 | 日度 | AKShare `futures_hog_info` |
| **宏观** | 冻肉收储/投放 | 政策干预 | 不定期 | 华储网公告 |
| **期货** | 生猪期货 LH | 预期定价 | 日度 | AKShare 期权接口 |

### 1.3 不同用户关注什么

| 用户角色 | 核心关注 | 使用频率 | 关键决策场景 |
|---------|---------|---------|------------|
| **养殖户** | 现在补栏/淘汰是否划算？ | 每周1-2次 | 看养殖利润 + 能繁母猪趋势，判断未来10个月供需 |
| **饲料商** | 未来几个月猪存栏量趋势？ | 每周1次 | 看能繁母猪 + 仔猪价格，预判饲料需求 |
| **屠宰企业** | 近期猪价走势？压栏风险？ | 每日1次 | 看出栏体重 + 猪价 + 屠宰量，判断供给节奏 |
| **期货投机者** | 猪周期当前位置？方向？ | 每日多次 | 看全链条指标，判断期货价格合理性 |

### 1.4 产品核心用户定位

**主目标用户：有投资决策需求的个人（期货投机者/养殖户）**。理由：
- 数据更新频率匹配（日度/周度够用）
- 愿意花时间看图表分析趋势
- 决策价值清晰（赚钱/亏钱的判断）
- 饲料商/屠宰企业更依赖内部数据，不是公开看板的目标群体

## 2. 数据源方案

### 2.1 一期数据源：AKShare（免费）

AKShare 生猪接口已统一更名为 `futures_hog_*` 前缀：

| 接口 | 数据内容 | 颗粒度 | 备注 |
|------|---------|--------|------|
| `futures_hog_info` | 全国/各省生猪价格（外三元、内三元、土杂猪）、玉米、豆粕 | 日度 | 数据源：zhujia.zhuwang.cc |
| `futures_hog_rank` | 各省生猪价格排行 | 日度 | |
| `futures_hog_core` | 能繁母猪存栏、生猪存栏、生猪出栏、猪肉产量 | 年度 | 起始2009年 |
| `futures_hog_cost` | 养殖成本相关数据 | 月度 | |
| `futures_hog_supply` | 猪肉批发价、冻猪肉储备、饲料原料、白条肉、生猪产能 | 月度 | |

**局限性**：
- 核心指标（能繁母猪）仅年度数据，颗粒度不够 → 需补充国家统计局月度数据
- 出栏体重、屠宰量等需付费数据源，一期不纳入
- AKShare 接口可能随数据源网站变动而失效，需异常监控

### 2.2 二期可扩展数据源

| 数据源 | 内容 | 获取方式 | 成本 |
|--------|------|---------|------|
| 国家统计局 | 月度能繁母猪存栏 | 网页抓取 / AKShare 宏观接口 | 免费 |
| MacroMicro | 生猪/能繁母猪存栏图表数据 | API（有限免费额度） | 免费/付费 |
| 博亚和讯 | 猪粮比、猪料比、白条猪鲜销率 | 付费 | 专业版 |
| 涌益咨询 | 出栏体重、屠宰量、能繁母猪高频 | 付费 | 机构级 |
| 大连商品交易所 | 生猪期货 LH 日线 | AKShare 期货接口 | 免费 |

### 2.3 数据更新频率设计

```
日更（cron daily 18:00）：生猪价格、玉米豆粕、期货价格
周更（cron weekly 周一）：猪粮比、养殖利润（周度汇总）
月更（cron monthly 1号）：能繁母猪存栏（统计局月度数据）
年更（cron yearly）：年度产量数据
```

## 3. 竞品分析摘要

### 3.1 现有产品

| 产品 | 定位 | 亮点 | 不足 |
|------|------|------|------|
| **博亚和讯** | 专业生猪数据平台 | 指标全面，行业权威 | 付费墙高，无免费可视化 |
| **农信互联猪联网** | 养猪产业链平台 | 数据覆盖广，物联网集成 | 面向B端猪场管理，非投资分析 |
| **涌益咨询** | 券商级高频数据 | 数据质量高，研报广泛引用 | 付费，无公开看板 |
| **MacroMicro** | 宏观经济图表 | 交互式图表，国际化 | 猪周期指标少，非专注 |
| **帆软农牧看板** | BI通用方案 | 模板化，快速部署 | 过于通用，缺乏猪周期专业逻辑 |

### 3.2 专业金融终端的启示

- **TradingView**：蜡烛图 + 叠加指标 + 多时间框架 + 十字光标联动，单图表信息密度极高
- **Wind/iFinD**：单图多指标叠加、频率换算、指标间运算，支持将不同频率数据统一展示
- **共同特点**：少页面、高信息密度、专业配色、平滑交互

### 3.3 我们的差异化

市面没有**免费的、面向个人投资者的猪周期专业分析看板**。这是机会。

## 4. 产品设计

### 4.1 设计理念

1. **一屏看懂周期位置** — 用户打开看板，5 秒内知道现在处于猪周期的哪个阶段
2. **信息密度优先** — 参考专业金融终端，不是"大字报式 BI 大屏"
3. **可下钻但不强制** — 首屏是结论，细节在交互后展开
4. **深色主题** — 金融终端标配，长时间查看不疲劳，价格涨跌配色直觉化（红涨绿跌）

### 4.2 信息架构：仅 1 个页面

不做多页导航。**一个长页面，3 个区域**，从上到下信息粒度递进。

### 4.3 各组件详细设计

#### 4.3.1 周期位置指示器（HEADER）

- 横向进度条，标注"繁荣→高峰→衰退→谷底→复苏"五个阶段
- 基于简化评分模型（v1 可用规则引擎，无需 ML）：
  - 能繁母猪同比变化（权重 40%）
  - 猪粮比位置（权重 30%）
  - 仔猪价格趋势（权重 30%）
- 指示器位置 = 加权评分映射到五阶段
- **v1 用固定规则，不做复杂模型** — 核心价值在图表，不在评分

#### 4.3.2 核心指标卡片（4 张）

每张卡片显示：
- 指标名称 + 更新时间
- 当前值（大字号）
- 环比变化（MoM）+ 同比变化（YoY）+ 小箭头/颜色
- 点击卡片 → 主图切换为该指标的详细视图

指标选择：
1. **生猪价格（外三元）** — 日度，最直接的关注点
2. **能繁母猪存栏** — 月度，核心领先指标
3. **仔猪价格** — 周度，补栏意愿信号
4. **养殖利润** — 周度（自算），盈亏判断

#### 4.3.3 主图：猪价 vs 能繁母猪双轴图

这是**整个看板的核心**。

- **上轴（左Y）**：生猪价格 + 生猪期货 LH 连续合约（虚线），单位 元/kg
- **下轴（右Y）**：能繁母猪存栏，万头，柱状图
- **叠加参考线**：猪粮比 5.5（蓝色虚线）和 7.0（绿色虚线）
- **交互**：
  - 时间范围滑块（dataZoom）拖拽缩放
  - 鼠标悬停十字光标，所有数据系列联动显示当前时间点数值
  - 图例点击切换显示/隐藏
- **设计要点**：猪价线在上方、能繁柱状在底部，视觉上形成"价高猪少→价低猪多"的直观对应

#### 4.3.4 猪粮比 & 养殖利润图

- 双线图，猪粮比 + 自算养殖利润
- 标注盈亏平衡线（猪粮比 5.5）
- 利润为负时填充浅红色区域，正值填充浅绿色
- 数据频率：周度

#### 4.3.5 成本结构图

- 玉米价格、豆粕价格、育肥猪配合饲料价格三线叠加
- 数据频率：周度
- 可叠加生猪价格做价差对比

#### 4.3.6 周期对比图

- 将当前周期的猪价走势与历史 3 轮周期重叠展示
- X 轴为"距周期谷底的月数"，而非绝对时间
- 帮助用户判断"当前走到哪个阶段了"
- 数据：至少需要 3 个完整周期（约 12 年），AKShare 数据从 2009 年起基本够用

#### 4.3.7 季节性热力图

- X 轴：1-12 月，Y 轴：年份
- 单元格颜色：当月猪价环比涨跌幅
- 底部汇总：每月平均涨跌概率
- 帮助用户把握季节性规律（如每年 Q1 通常淡季、Q3-Q4 旺季）

#### 4.3.8 数据明细表（可折叠）

- 默认收起，点击展开
- 最近 30 行数据，列：日期、猪价、玉米、豆粕、猪粮比、养殖利润
- 导出 CSV 按钮
- **不作为核心功能**，只是兜底

### 4.4 交互设计

| 交互 | 行为 | 备注 |
|------|------|------|
| 时间范围 | 主图底部 dataZoom 滑块 + 快捷按钮（1Y/3Y/5Y/ALL） | 默认 3Y |
| 指标卡片点击 | 主图切换为该指标详细视图 | 再次点击或点其他卡片恢复默认 |
| 图例点击 | 显示/隐藏数据系列 | ECharts 原生支持 |
| 十字光标 | 鼠标悬停显示时间点所有数值 | ECharts tooltip 联动 |
| 数据导出 | 表格区域导出 CSV | 生成后下载 |
| 页面刷新 | 数据自动加载最新，无需手动 | 后端 cron 采集，前端读 DB |

### 4.5 视觉规范

- **主题**：深色（#1a1a2e 背景，#16213e 卡片）
- **涨跌色**：红涨绿跌（中国 A 股惯例）
- **字体**：数字用等宽字体（JetBrains Mono / Source Code Pro），保证对齐
- **图表配色**：ECharts 深色主题，主色调用暖色系（猪价橙色、能繁蓝色、利润绿色/红色）
- **响应式**：适配桌面端（1920px）为主，平板端（1024px）可用，手机端不优先

## 5. 技术选型

### 5.1 推荐方案：FastAPI + ECharts（纯前后端分离）

| 组件 | 选型 | 理由 |
|------|------|------|
| **后端** | FastAPI | 内存占用 ~50MB，异步支持，自动 API 文档 |
| **前端图表** | ECharts（CDN引入） | Canvas 渲染快 10x，原生 K 线/dataZoom，金融场景最佳 |
| **前端框架** | 纯 HTML + Vanilla JS（或极简 HTMX） | 单页面，无需 SPA 框架，减少复杂度 |
| **数据存储** | SQLite | 数据量极小（<1MB），零运维 |
| **数据采集** | Python + AKShare + cron | 成熟稳定 |
| **部署** | systemd + Nginx 反代 | 轻量，2-4GB VPS 轻松运行 |

### 5.2 为什么不用 Dash/Streamlit

| 维度 | Dash/Streamlit | FastAPI + ECharts |
|------|---------------|-------------------|
| 内存占用 | 200MB-2GB | ~50MB（FastAPI）+ 浏览器端渲染 |
| 图表交互性 | Plotly 够用但非金融专业级 | ECharts 原生 dataZoom/candlestick/tooltip联动 |
| 自定义 UI | 受限于框架组件 | 完全自由，像素级控制 |
| 部署复杂度 | Gunicorn worker 管理 | uvicorn 单进程即可 |
| 多用户并发 | Dash 每用户 session 占内存 | 无状态 API，浏览器端渲染 |

**核心原因**：Dash/Plotly 是通用数据科学工具，ECharts 是专业可视化库。猪周期看板的**图表交互质量**是核心竞争力，ECharts 在这方面领先。

### 5.3 架构图

```
cron (定时采集)
  │
  ▼
Python 采集脚本 (AKShare API)
  │
  ▼
SQLite (pig_cycle.db)
  │
  ▼
FastAPI (REST API: GET /api/price, /api/sow, /api/profit...)
  │
  ▼
Nginx → 静态前端 (HTML + ECharts CDN + vanilla.js)
         │
         ▼
       浏览器渲染
```

### 5.4 API 设计（6 个接口）

| 端点 | 返回 | 说明 |
|------|------|------|
| `GET /api/overview` | 4张卡片的当前值 + 环比/同比 | JSON |
| `GET /api/price?range=3y` | 猪价 + 期货 + 能繁母猪时序 | JSON |
| `GET /api/profit?range=3y` | 猪粮比 + 养殖利润时序 | JSON |
| `GET /api/cost?range=3y` | 玉米/豆粕/饲料价格时序 | JSON |
| `GET /api/cycle-compare` | 历史周期重叠数据 | JSON |
| `GET /api/seasonality` | 月度涨跌概率矩阵 | JSON |

所有接口返回 JSON，前端 ECharts 直接消费。无需 WebSocket，数据更新频率低（日更/周更）。
