## 2. 定时更新方案设计

### 2.1 更新频率策略

根据各类数据的发布规律，设计如下更新策略：

| 数据频率 | cron 时间 | 数据内容 | 自动化程度 |
|---------|----------|---------|-----------|
| **日度** | 每日 19:30（A股收盘后） | 外三元、内三元、土杂猪、玉米、豆粕价格 | 全自动（AKShare） |
| **周度** | 每周日 20:00 | 猪粮比、养殖利润估算、出栏体重（若有）、告警检查 | 半自动（AKShare + 公式估算） |
| **月度** | 每月 15 日 19:00（官方数据发布窗口） | 能繁母猪存栏、生猪存栏、出栏量、饲料产量 | 半自动（AKShare 或网页抓取） |
| **季度** | 每季末月 25 日 19:00 | 统计局宏观数据、上市公司季报汇总 | 手动/半自动 |

> **说明**：月度数据官方通常在月中旬发布（如农业农村部每月 15 日前后），但发布时间有波动。设为每月 15 日和 20 日各运行一次，取最新数据。

#!/usr/bin/env python3
"""日度数据采集"""
import sys
sys.path.insert(0, '/opt/pig-tracker')

import logging
from collector.akshare_collector import AKShareCollector
from storage.repository import Repository

logging.basicConfig(
    filename='logs/daily.log',
    level=logging.INFO,
    format='%(asctime)s %(levelname)s %(message)s'
)
logger = logging.getLogger(__name__)

def main():
    repo = Repository('/opt/pig-tracker/pig_cycle.duckdb')
    collector = AKShareCollector(repo)

    try:
        rows = collector.collect_daily_prices()  # 外三元、内三元、玉米、豆粕
        logger.info(f"日度采集成功，新增 {rows} 行")
    except Exception as e:
        logger.error(f"日度采集失败: {e}")
        # 触发 Telegram 告警（连续失败计数）
        raise

if __name__ == '__main__':
    main()

#### `jobs/weekly_analyze.py` 核心逻辑

```python
#!/usr/bin/env python3
"""周度分析 + 报告生成"""
import sys
sys.path.insert(0, '/opt/pig-tracker')

from analyzer.cycle_scorer import CycleScorer
from analyzer.alert_rules import AlertEngine
from reporter.weekly_report import WeeklyReportGenerator
from storage.repository import Repository
import logging

logging.basicConfig(filename='logs/weekly.log', level=logging.INFO)
logger = logging.getLogger(__name__)

def main():
    repo = Repository('/opt/pig-tracker/pig_cycle.duckdb')
    scorer = CycleScorer()
    alerter = AlertEngine(repo)
    reporter = WeeklyReportGenerator(repo)

    # 1. 计算当前周期评分
    current_data = repo.get_latest_indicators()
    score_result = scorer.compute_overall_score(current_data)
    logger.info(f"周期评分: {score_result['overall_score']} ({score_result['phase']})")

    # 2. 检查告警规则
    triggered = alerter.check_all(current_data)
    for alert in triggered:
        alerter.send_telegram(alert)  # Telegram 推送

    # 3. 生成周报 Markdown
    report_path = reporter.generate(score_result, current_data)
    logger.info(f"周报已生成: {report_path}")

    # 4. 推送周报摘要到 Telegram
    summary = f"🐷 猪周期周报\n评分: {score_result['overall_score']}/100\n阶段: {score_result['phase']}\n详见: {report_path}"
    alerter.send_telegram_text(summary)

if __name__ == '__main__':
    main()
```

### 2.4 数据采集失败处理策略

```
采集失败处理流程（优先级从高到低）：

[一级] AKShare 单接口失败
  → 记录 warning 到 collection_log
  → 该指标标记为 NULL，继续其他指标
  → 3 次连续失败 → Telegram 告警 "⚠️ {指标} 连续采集失败"

[二级] AKShare 全局不可用
  → 降级模式：跳过自动采集，标记 "数据缺失"
  → 尝试备用数据源（国家统计局网页）
  → Telegram 告警 "🔴 AKShare 不可用，进入降级模式"

[三级] 网络超时
  → 指数退避重试：3s → 9s → 27s
  → 3 次重试后仍失败 → 按一级处理

[四级] 数据异常值（涨跌幅 >20%）
  → 记录 warning，标注 "待验证"
  → 不自动修正，在周报中显示 "⚠️ 数据异常，请人工确认"
```

### 2.5 与 OpenClaw 的集成方式

**结论：推荐使用 OpenClaw cron（而非系统 cron）**

理由：
1. OpenClaw cron 与 OpenClaw 生态深度集成，告警可直接通过 OpenClaw 推送
2. 无需配置系统级 cron 权限
3. 任务状态可在 OpenClaw 管理界面查看

**集成方式 A：OpenClaw 内置 cron（推荐）**
```yaml
# openclaw cron 配置（通过 openclaw cli 或配置文件）
pig-cycle-daily:
  command: python /opt/pig-tracker/jobs/daily_collect.py
  schedule: "30 19 * * 1-5"
  notify: telegram
  log: /opt/pig-tracker/logs/daily.log

pig-cycle-weekly:
  command: python /opt/pig-tracker/jobs/weekly_analyze.py
  schedule: "0 20 * * 0"
  notify: telegram
  log: /opt/pig-tracker/logs/weekly.log
```

**集成方式 B：系统 cron + OpenClaw 通知（备选）**
- 如果 OpenClaw cron 不可用，用系统 cron 调度
- 告警通过 OpenClaw Agent Skill 调用 Telegram API

## 3. 可视化方案设计

| **matplotlib 生成图片** | 部署简单（纯 Python），可嵌入 Markdown，样式可控 | 需要定期重新生成（非实时） | ⭐⭐⭐⭐⭐ |
| Chart.js 静态 HTML | 交互性好，支持缩放/悬停 | 需要前端服务或托管，纯静态不够灵活 | ⭐⭐⭐ |
| Plotly | 交互好，Python API | 需要 `plotly` 库，部署略复杂 | ⭐⭐ |
| Markdown 表格/ASCII | 无需依赖，最简单 | 可读性差，无法展示趋势 | ⭐⭐ |

**推荐理由**：
- 轻量级，服务器上 `pip install matplotlib` 即可
- 与 Markdown 周报无缝集成：`![](chart.png)`
- 无需启动 Web 服务
- 可通过 cron 定期重新生成（如每周日生成周报图表）

# visualization/dashboard.py
import matplotlib.pyplot as plt
import numpy as np

def generate_cycle_dashboard(score_result: dict, output_path: str):
    """
    生成周期位置仪表盘图：
    - 中央：综合评分（0-100）圆形仪表
    - 周围：6个子指标条形图
    - 底部：当前阶段文字标注
    """

    fig = plt.figure(figsize=(14, 8))

    # 中央仪表盘（极坐标）
    ax_outer = fig.add_subplot(121, polar=True)

    # 仪表盘刻度（0-100，分成 6 段对应 P0-P5）
    angles = np.linspace(0, np.pi, 6)
    colors = ['#27ae60', '#f1c40f', '#e67e22', '#e74c3c', '#c0392b', '#8e44ad']

    # 外圈：评分刻度
    # 填充色块表示当前评分位置
    score = score_result['overall_score']  # 0-100
    theta = np.linspace(0, np.pi * score / 100, 100)
    ax_outer.fill_between(theta, 0, 1, color='#27ae60', alpha=0.3)

    ax_outer.set_xticks([])
    ax_outer.set_yticks([])
    ax_outer.set_title(f'周期评分\n{score}/100', fontsize=16, fontweight='bold', pad=20)

    # 右图：子指标雷达图
    ax_radar = fig.add_subplot(122, polar=True)
    sub_scores = score_result['sub_scores']
    indicators = list(sub_scores.keys())
    values = list(sub_scores.values())

    angles_radar = np.linspace(0, 2*np.pi, len(indicators), endpoint=False).tolist()
    values_plot = values + [values[0]]  # 闭合
    angles_radar = angles_radar + [angles_radar[0]]

    ax_radar.plot(angles_radar, values_plot, 'o-', linewidth=2)
    ax_radar.fill(angles_radar, values_plot, alpha=0.25)
    ax_radar.set_xticks(angles_radar[:-1])
    ax_radar.set_xticklabels(indicators, fontsize=8)
    ax_radar.set_ylim(0, 100)
    ax_radar.set_title('子指标评分', fontsize=12)

    plt.tight_layout()
    plt.savefig(output_path, dpi=150)
    plt.close()
    print(f"周期仪表盘已保存: {output_path}")

# visualization/mom_changes.py

def generate_mom_changes(repo, output_path: str):
    """
    生成各指标环比变化图：
    - 横向柱状图
    - 绿色 = 环比上升（对周期位置的影响）
    - 红色 = 环比下降
    - 0 线居中
    """
    latest = repo.get_latest_indicators()
    prev = repo.get_previous_period_indicators()

    indicators = ['pig_price', 'pig_corn_ratio', 'profit', 'breeding_sow']
    changes = [(latest[i] - prev[i]) / prev[i] * 100 for i in indicators]

    colors = ['#27ae60' if c > 0 else '#e74c3c' for c in changes]

    fig, ax = plt.subplots(figsize=(10, 5))
    y_pos = np.arange(len(indicators))
    ax.barh(y_pos, changes, color=colors)
    ax.axvline(x=0, color='black', linewidth=0.8)
    ax.set_yticks(y_pos)
    ax.set_yticklabels(['猪价', '猪粮比', '养殖利润', '能繁母猪'])
    ax.set_xlabel('环比变化 (%)')
    ax.set_title('各核心指标环比变化（绿色=上升/利好，红色=下降/利空）')
    plt.tight_layout()
    plt.savefig(output_path, dpi=150)
    plt.close()
