## 一、OpenAI — Deep Research

### 1.1 产品概述

OpenAI 于 **2025年2月2日** 正式推出 Deep Research 功能，定位为"可在互联网上针对复杂任务进行多步骤研究的代理功能"。底层模型为 **o3**（推理模型），通过端到端强化学习训练，具备复杂任务拆解与多步推理能力。2025年7月，Deep Research 与 Operator 合并为统一的 **ChatGPT Agent**，获得视觉浏览器能力。

### 1.2 产品形态和入口

- **入口方式**：ChatGPT 对话界面中的**独立模式选项**，非默认模式
- **触发方式**：手动选择 "Deep Research" 模式
- **文件支持**：可处理用户上传的文件，搜索公共网络或特定站点
- **两种版本**：标准版（完整深度研究）和轻量版（faster, simpler queries）
- **API**：Deep Research API 已上线，开发者可集成

### 1.3 交互方式

1. 用户切换到 Deep Research 模式
2. 输入研究问题（自然语言描述）
3. 系统展示研究计划，用户可调整方向
4. **实时进度展示**：显示当前搜索和阅读的页面
5. 执行时间：**5-73 分钟**（一个案例中查阅了 29+ 来源）
6. **支持中途干预**：可在研究过程中提供反馈
7. 最终生成结构化研究报告

### 1.4 搜索和信息收集机制

- **并行搜索**：可同时搜索数百个网页资源
- **多源覆盖**：公共网络搜索 + 用户指定的特定站点 + 上传文件
- **信息筛选**：基于 o3 推理能力自动评估信息相关性和可靠性
- **多轮迭代**：根据搜索结果动态调整搜索策略
- **视觉浏览器**：ChatGPT Agent 获得视觉浏览器能力后，可"go even deeper and broader"

### 1.5 推理/研究流程

- **多步骤规划**：o3 模型自主拆解研究任务，制定搜索计划
- **并行执行**：多线程同时搜索和阅读
- **自我反思**：在执行过程中评估信息充分性，发现缺口则补充搜索
- **信息综合**：将收集到的信息通过推理进行整合分析

### 1.6 输出格式

- **结构化报告**：Markdown 格式，含摘要、正文、结论
- **完整引用列表**：每项信息标注来源 URL
- **可读性强**：段落清晰，逻辑连贯
- **导出支持**：可复制为 Markdown 或导出为文件

### 1.7 技术架构

- **底层模型**：o3（推理模型），通过端到端强化学习训练
- **Agent 架构**：单 Agent 多线程，非多 Agent 集群
- **关键能力**：Tool Use（搜索工具）+ 推理（reasoning）+ 信息整合
- **2025年7月**：与 Operator 合并为统一 ChatGPT Agent

### 1.8 定价和可用性

| 套餐 | Deep Research 配额 | 价格 |
|------|-------------------|------|
| Free | 5次/月 | 免费 |
| Plus（$20/月） | 25次/月 | 付费 |
| Pro（$100-200/月） | 100-120次/月 | 付费 |
| Team/Enterprise | 25次/月 | 企业定价 |

## 二、Google — Gemini Deep Research

### 2.1 产品概述

Google 的 Gemini Deep Research 最早于 **2024年12月** 随 Gemini 2.0 发布，是全球首个由科技巨头推出的"深度研究"产品。定位为"你的个人研究助手"，可自动规划、搜索、分析并生成研究报告。2025年进一步升级：Enterprise 版可结合 Gmail/Drive/Chat 内部数据，AI Ultra 计划支持图表和交互式模拟器，ADK 向开发者开放。

### 2.2 产品形态和入口

- **入口方式**：Google Gemini（gemini.google.com）中的**独立功能**
- **触发方式**：用户手动选择 "Deep Research" 模式
- **支持范围**：150 个国家、45+ 语言
- **开发者 API**：gemini/deep-research-pro-preview（单次约 $3）

### 2.3 交互方式

1. 用户输入研究主题
2. 系统自动生成研究计划（可查看和调整）
3. **动态进度展示**：实时显示搜索过程和发现的来源
4. 执行时间：**数分钟到数十分钟**
5. 研究完成后通知用户（即使离开页面）
6. 历史研究可在侧边栏查看

### 2.4 搜索和信息收集机制

- **Google 搜索 + Gemini 推理**：基于 Google 搜索基础设施
- **多源整合**：网页 + 学术论文 + 企业内部数据（Enterprise 版）
- **信息评估**：批判性评估信息，识别关键主题和矛盾点
- **多轮迭代**：发现信息缺口时自动补充搜索

### 2.5 推理/研究流程

- **多步强化学习**：减少幻觉
- **信息验证**：主动交叉验证不同来源
- **逻辑结构化**：按逻辑组织报告
- **批判性评估**：识别信息矛盾

### 2.6 输出格式

- **结构化研究报告**：含摘要、详细分析、结论
- **来源引用**：行内引用 + 末尾列表
- **Google Docs 集成**：公式转为 TeX/LaTeX 格式
- **可视化**：AI Ultra 用户可获得图表、交互式模拟器

### 2.7 技术架构

- **底层模型**：Gemini 2.0/2.5（多模态）
- **Agent 架构**：沙箱执行环境，隔离容器运行
- **Google ADK**：Agent Development Kit 供开发者使用
- **Gemini Interactions API**：多轮交互式研究

### 2.8 定价和可用性

| 套餐 | Deep Research 配额 | 价格 |
|------|-------------------|------|
| Free | 有使用次数限制 | 免费 |
| Google AI Pro（$19.99/月） | 完整功能 + 2TB 存储 | 付费 |
| AI Ultra | 可视化报告 + 交互模拟器 | 更贵 |
| Enterprise | 企业级（Gmail/Drive集成） | 企业定价 |
| API | 约 $3/次 | 开发者 |

## 三、Anthropic — Claude Research 能力

### 3.1 产品概述

Anthropic **未推出独立 "Deep Research" 产品**，而是通过 Claude 的组合能力实现研究功能：**Extended Thinking + Web Search Tool + Computer Use + Tool Use + MCP**。2026年3月 Computer Use 进入预览。Anthropic 的架构哲学是"不给研究按钮，让能力自然组合"。

### 3.2 产品形态和入口

- **入口方式**：无独立入口，是 Claude 对话能力的**自然组合**
- **核心能力组合**：
  - **Extended Thinking**：深度推理模式（Claude 3.7 Sonnet 引入，Sonnet 4/4.6 增强）
  - **Web Search Tool**：内置网页搜索，Claude 自主判断是否需要搜索
  - **Computer Use**：自主操作浏览器、文件、开发工具（2026.3 预览）
  - **MCP**：开放协议，连接 Slack/Drive/Notion 等外部工具
  - **Tool Search**：可搜索数千个可用工具
  - **Web Fetch**：直接抓取网页和 PDF 全文

### 3.3 交互方式

1. 用户在 Claude 对话中提问
2. Claude 根据问题复杂度决定是否需要搜索
3. **Extended Thinking**：复杂问题自动启用深度推理
4. **多轮工具调用**：搜索 + 阅读 + 操作
5. **过程可见**：用户可看到 Claude 的思考步骤透明度
6. 无固定等待时间，取决于问题复杂度

### 3.4 搜索和信息收集机制

- **Anthropic 执行的搜索**：内置 Web Search Tool
- **自主判断**：Claude 推理判断是否需要搜索
- **MCP 生态**：连接外部工具和数据源
- **Web Fetch**：直接抓取网页/PDF 全文
- **信息评估**：通过 Extended Thinking 深度验证

### 3.5 推理/研究流程

- **三层策略**：Web Search（简单事实）→ Extended Thinking（复杂分析）→ Research 模式（综合）
- **多 Agent 架构**（开发者层）：Planner → Searcher → Reader → Writer
- **Long-running Claude**：支持长时间科学研究任务
- **API 精细控制**：可配置 thinking effort 级别

### 3.6 输出格式

- **引用标注**：行内引用 + 来源列表
- **无固定模板**：输出格式灵活

## 四、Perplexity — Pro Search / Deep Research

### 4.1 产品概述

Perplexity AI 是以搜索为核心的 AI 产品。Deep Research 使用**有向无环图（DAG）和递归**编排研究工作流，执行 **20-50 次定向查询**，阅读数百个来源后生成综合报告。多模型可选（GPT-5.1、Claude Sonnet 4.5 等）。

### 4.2 产品形态和入口

- **入口方式**：内置在 Perplexity 主产品中
- **三种模式**：Focus（标准）→ Pro Search（多步骤，先追问）→ Deep Research（完整报告）
- **触发方式**：用户选择模式；Pro Search 会先提问澄清需求

### 4.3 交互方式

1. 用户输入问题
2. **Pro Search 追问**：系统先提问以澄清范围和重点
3. 根据反馈确定搜索策略
4. **多轮搜索**：20-50 次定向查询，实时展示进度
5. 执行时间：Pro Search 约 **1-3 分钟**，Deep Research 更长

### 4.4 搜索和信息收集机制

- **DAG 图编排**：有向无环图 + 递归编排研究工作流
- **多源并行**：网页、学术、新闻、社交媒体
- **PDF 阅读**：基于检索的解析（retrieval-based parsing）+ 引用定位
- **质量筛选**：优先选择权威来源

### 4.5 推理/研究流程

- **多轮查询系统**（multi-pass querying）：AI 不满足于一次搜索
- **查询聚类**：自动聚类相关问题
- **自适应搜索**：根据初步结果调整策略

### 4.6 输出格式

- **引用式答案**：[1][2][3] 编号引用，可直接链接
- **来源卡片**：侧边栏完整列表
- **社区 Spaces**：用户可贡献搜索结果
- **分享链接**：可生成

### 4.7 技术架构

- **搜索增强生成（RAG）**：搜索优先（Search-first）
- **Graph-based Agent**：DAG 图编排研究工作流
- **多模型**：GPT-5.1、Claude Sonnet 4.5 等可选
- **实时索引**：维护实时网页索引

### 4.8 定价和可用性

| 套餐 | 深度研究能力 | 价格 |
|------|------------|------|
| Free | 标准 + 5次 Pro/天 | 免费 |
| Pro（$20/月） | 300+ Pro/天 + Deep Research + 10K credits | 付费 |
| Enterprise | 企业级 | 企业定价 |

## 十三、技术架构深度对比分析

### 13.1 三大技术路线

**路线一：推理模型驱动（Reasoning-First）**
- **代表**：OpenAI（o3）、Anthropic（Extended Thinking）、智谱（GLM-Z1-Rumination）、xAI（Heavy/CoT）
- **特点**：以推理模型为核心，通过深度推理链来规划和执行研究
- **优势**：研究质量高，推理能力强
- **劣势**：成本高、速度慢、token 消耗大

**路线二：Agent 架构驱动（Agent-First）**
- **代表**：Google（ADK/沙箱Agent）、Microsoft（双Agent编排）、Kimi（Agent集群）、百度（4阶段闭环）
- **特点**：以 Agent 框架为核心，通过多步骤规划和工具调用完成研究
- **优势**：灵活性强，可扩展性好
- **劣势**：需要精心设计的编排逻辑

**路线三：搜索增强驱动（Search-First）**
- **代表**：Perplexity（DAG图编排）、百度（搜索生态）
- **特点**：以搜索为核心，AI 生成为辅助
- **优势**：速度快、信息新鲜、引用精准
- **劣势**：深度推理能力有限

### 13.2 自训练 vs 通用模型

| 策略 | 代表 | 优势 | 劣势 |
|------|------|------|------|
| **自训练研究模型** | 阿里 Tongyi 30B-A3B（MoE） | 工具使用能力强、成本极低（仅激活3.3B）、全开源 | 需要大量算力和数据 |
| **通用推理模型** | OpenAI o3、Claude 4、GLM-Z1 | 研究质量高、通用性强 | 成本高、token消耗大 |
| **混合策略** | Google Gemini、字节豆包、百度千帆 | 兼顾质量和成本 | 架构复杂度更高 |

### 13.3 关键技术趋势

1. **Test-time Scaling**：通过增加推理步骤提升研究质量（阿里 IterResearch、OpenAI o3）
2. **Interactive Scaling**：训练模型处理更多工具调用（MiroThinker，300-400次调用）
3. **并行化**：Kimi Agent 集群（100 Agent）代表"用算力换质量"趋势
4. **开源追赶**：阿里 Tongyi 开源模型已达 SOTA，登顶 DeepResearch Bench
5. **Research + Operator 融合**：智谱"边想边干"、OpenAI ChatGPT Agent 合并 DR+Operator
6. **企业搜索**：Microsoft（Graph）和 Google（Gmail/Drive）的企业级研究是差异化方向
7. **实时数据**：xAI 的 X 平台数据是独特信息源
8. **隐私优先**：Mistral 的 GDPR 合规是欧洲市场差异化
9. **DAG 编排**：Perplexity 使用有向无环图编排研究工作流，20-50次定向搜索
10. **中国免费化**：百度、阿里、智谱、字节均免费提供深度研究，与 OpenAI $100-200/月 对比鲜明
