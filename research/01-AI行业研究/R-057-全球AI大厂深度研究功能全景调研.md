## 一、OpenAI — Deep Research

### 1.1 产品概述

OpenAI 于 **2025年2月2日** 正式推出 Deep Research 功能，定位为"可在互联网上针对复杂任务进行多步骤研究的代理功能"。底层模型为 **o3**（推理模型），通过端到端强化学习训练，具备复杂任务拆解与多步推理能力。

### 1.2 产品形态和入口

- **入口方式**：ChatGPT 对话界面中的**独立模式选项**，非默认模式
- **触发方式**：用户在 ChatGPT 中手动选择 "Deep Research" 模式，或系统自动识别复杂研究任务
- **文件支持**：可处理用户上传的文件，搜索公共网络或特定站点
- **两种版本**：标准版（完整深度研究）和轻量版（faster, simpler queries）

### 1.3 交互方式

1. 用户切换到 Deep Research 模式
2. 输入研究问题（自然语言描述）
3. 系统展示研究计划，用户可调整方向
4. **实时进度展示**：显示当前搜索和阅读的页面
5. 执行时间：**5-30 分钟**（取决于研究复杂度）
6. **支持中途干预**：可在研究过程中提供反馈
7. 最终生成结构化研究报告

### 1.4 搜索和信息收集机制

- **并行搜索**：可同时搜索数百个网页资源
- **多源覆盖**：公共网络搜索 + 用户指定的特定站点 + 上传文件
- **信息筛选**：基于 o3 推理能力自动评估信息相关性和可靠性
- **多轮迭代**：根据搜索结果动态调整搜索策略

### 1.5 推理/研究流程

- **多步骤规划**：o3 模型自主拆解研究任务，制定搜索计划
- **并行执行**：多线程同时搜索和阅读
- **自我反思**：在执行过程中评估信息充分性，发现缺口则补充搜索
- **信息综合**：将收集到的信息通过推理进行整合分析

### 1.6 输出格式

- **结构化报告**：Markdown 格式，含摘要、正文、结论
- **完整引用列表**：每项信息标注来源 URL
- **可读性强**：段落清晰，逻辑连贯
- **导出支持**：可复制为 Markdown 或导出为文件

### 1.7 技术架构

- **底层模型**：o3（推理模型），通过端到端强化学习训练
- **Agent 架构**：单 Agent 多线程，非多 Agent 集群
- **关键能力**：Tool Use（搜索工具）+ 推理（reasoning）+ 信息整合
- **2025年7月**：OpenAI 发布 ChatGPT Agent，整合 Deep Research 到更广泛的 Agent 生态中

### 1.8 定价和可用性

| 套餐 | Deep Research 配额 | 价格 |
|------|-------------------|------|
| Plus（$20/月） | 10次/月（标准版） | 付费 |
| Pro（$200/月） | 250次/月（125标准+125轻量） | 付费 |
| Free | 不支持 | — |
| Team/Enterprise | 按团队方案 | 付费 |

## 二、Google — Gemini Deep Research

### 2.1 产品概述

Google 的 Gemini Deep Research 最早于 **2024年12月** 随 Gemini 2.0 发布，是全球首个由科技巨头推出的"深度研究"产品。定位为"你的个人研究助手"，可自动规划、搜索、分析并生成研究报告。2025年进一步升级，扩展至企业级（Gemini Enterprise）和开发者 API。

### 2.2 产品形态和入口

- **入口方式**：Google Gemini（gemini.google.com）中的**独立功能**
- **触发方式**：用户在 Gemini 中手动选择 "Deep Research" 模式
- **2025年升级**：
  - Gemini Enterprise 版本：可搜索企业内部信息（Google Drive、Gmail 等）
  - Gemini API：开发者可调用 Deep Research Agent API
  - Google ADK（Agent Development Kit）：官方提供构建深度研究 Agent 的框架

### 2.3 交互方式

1. 用户输入研究主题
2. 系统自动生成研究计划（可查看和调整）
3. **动态进度展示**：实时显示搜索过程和发现的来源
4. 执行时间：**数分钟到数十分钟**
5. 研究完成后通知用户（即使离开页面）
6. 历史研究可在侧边栏查看

### 2.4 搜索和信息收集机制

- **Google 搜索 + Gemini 推理**：基于 Google 强大的搜索基础设施
- **多源整合**：网页搜索 + 学术论文 + 企业内部数据（Enterprise 版）
- **信息评估**：Gemini 批判性评估信息，识别关键主题和不一致之处
- **多轮迭代**：发现信息缺口时自动补充搜索

### 2.5 推理/研究流程

- **多步强化学习**：通过多步强化学习训练，减少幻觉
- **信息验证**：主动交叉验证不同来源的信息
- **逻辑结构化**：将研究结果按逻辑结构组织
- **批判性评估**：识别信息矛盾和不一致之处

### 2.6 输出格式

- **结构化研究报告**：含摘要、详细分析、结论
- **来源引用**：行内引用 + 末尾引用列表
- **Google Docs 集成**：可导出到 Google Docs
- **可视化图表**：支持在报告中嵌入数据可视化

### 2.7 技术架构

- **底层模型**：Gemini 2.0/2.5（多模态）
- **Agent 架构**：沙箱执行环境，每次研究在隔离容器中运行
- **关键创新**：
  - Made by Google Agent 框架
  - Google ADK（Agent Development Kit）供开发者使用
  - Gemini Interactions API 支持多轮交互式研究
- **安全机制**：沙箱执行，安全操作优先

### 2.8 定价和可用性

| 套餐 | Deep Research 配额 | 价格 |
|------|-------------------|------|
| Free | 有使用次数限制 | 免费 |
| Gemini Advanced（$20/月） | 更多次数 | 付费 |
| Google One AI Premium | 包含 | 付费 |
| Gemini Enterprise | 企业级 | 企业定价 |
| Gemini API | 按 token 计费 | 开发者 |

## 三、Anthropic — Claude Research 能力

### 3.1 产品概述

Anthropic 并未推出名为 "Deep Research" 的独立功能，而是通过 **Claude 的组合能力**（Extended Thinking + Web Search + Tool Use）实现研究功能。2025年，Anthropic 发布了详细的多 Agent 研究系统架构文档，展示了如何用 Claude 构建深度研究 Agent。

### 3.2 产品形态和入口

- **入口方式**：无独立入口，是 Claude 对话能力的**自然组合**
- **核心能力组合**：
  - **Web Search Tool**：2025年推出的内置网页搜索工具
  - **Extended Thinking**：深度推理模式（o1/o3 类似）
  - **Tool Use**：丰富的工具调用能力
  - **MCP（Model Context Protocol）**：开放协议，连接外部工具和数据源
- **Claude Research 模式**：claude.ai 对话中提供独立的 Research 模式（2025年推出）

### 3.3 交互方式

1. 用户在 Claude 对话中提问
2. Claude 根据问题复杂度决定是否需要搜索
3. **Extended Thinking 模式**：复杂问题自动启用深度推理
4. **Web Search**：需要外部信息时自动调用搜索工具
5. **多轮工具调用**：可能多次搜索 + 阅读网页
6. **实时过程可见**：用户可看到 Claude 的搜索和思考过程
7. 无固定的"等待时间"，取决于问题复杂度

### 3.4 搜索和信息收集机制

- **Anthropic 执行的搜索**：通过 Anthropic 内置的 Web Search Tool
- **搜索范围**：公共网络（Anthropic 控制搜索质量和安全性）
- **工具搜索（Tool Search）**：Claude 可搜索数千个可用工具而不消耗上下文窗口
- **MCP 生态**：可连接 Slack、Google Drive、Notion 等外部工具
- **信息评估**：通过 Extended Thinking 进行深度推理和验证

### 3.5 推理/研究流程

- Anthropic 官方推荐的三层研究策略：
  - **Web Search**：适合简单事实查询（1-2次工具调用）
  - **Extended Thinking**：适合需要深度推理的复杂分析
  - **Research 模式**：综合搜索 + 推理，适合深度研究
- **多 Agent 架构**（开发者层面）：
  - Planner Agent → Searcher Agent → Reader Agent → Writer Agent
  - Anthropic 官方博客详细介绍了这一架构
- **Long-running Claude**：支持长时间运行的科学研究任务

### 3.6 输出格式

- **对话式回答**：自然语言回答，附搜索来源
- **引用标注**：行内引用 + 来源列表
- **结构化**：用户可要求特定格式（报告、列表、表格等）
- **无固定模板**：输出格式灵活，取决于用户需求

### 3.7 技术架构

- **底层模型**：Claude 4 系列（Opus/Sonnet）
- **关键技术**：
  - Extended Thinking：类似 o1/o3 的推理链
  - Tool Use：原生工具调用能力
  - MCP（Model Context Protocol）：开放工具协议
  - Tool Search：动态发现和调用工具
- **架构哲学**：不给 Claude "研究"按钮，而是让能力自然组合
- **多 Agent 支持**：官方提供了多 Agent 研究系统构建指南

### 3.8 定价和可用性

| 套餐 | 研究能力 | 价格 |
|------|---------|------|
| Free | 基础对话，有限搜索 | 免费 |
| Pro（$20/月） | Web Search + Extended Thinking | 付费 |
| Max（$100/月/$200/月） | 更高配额 | 付费 |
| API | 按 token 计费 | 开发者 |
| Team/Enterprise | 按团队方案 | 企业定价 |

## 四、Perplexity — Pro Search / Deep Research

### 4.1 产品概述

Perplexity AI 是以搜索为核心的 AI 产品，其深度研究能力随产品迭代不断升级。最初以 "Pro Search"（多步骤搜索）著称，后续推出 "Deep Research" 功能，生成更深入的研究报告。

### 4.2 产品形态和入口

- **入口方式**：内置在 Perplexity 主产品中，非独立产品
- **两种模式**：
  - **Focus 模式**：标准搜索回答
  - **Pro Search**：多步骤深度搜索（先问澄清问题，再搜索）
  - **Deep Research**：更深入的多步骤研究，生成完整报告
- **触发方式**：用户可选择模式，Pro Search 会先向用户提问以明确需求

### 4.3 交互方式

1. 用户输入问题
2. **Pro Search 特点**：系统先提问以澄清研究范围和重点
3. 根据用户反馈确定搜索策略
4. **多轮搜索**：执行多轮搜索并实时展示进度
5. 执行时间：Pro Search 约 **1-3 分钟**，Deep Research 更长
6. 用户可在过程中回答澄清问题

### 4.4 搜索和信息收集机制

- **多源并行搜索**：同时搜索多个来源
- **智能追问**：Pro Search 会根据问题复杂度主动向用户追问
- **来源丰富**：网页、学术论文、新闻、社交媒体等
- **实时性**：搜索结果包含最新信息
- **社区贡献**：Perplexity 的搜索结果部分来自用户社区贡献（"Spaces"）

### 4.5 推理/研究流程

- **SQuAD 流程**：Search → Question → Answer → Digest
- **多步骤执行**：不是一次性搜索，而是多轮搜索+综合
- **用户协作**：通过追问机制让用户参与研究方向的确定
- **自适应搜索**：根据初步结果调整后续搜索策略

### 4.6 输出格式

- **引用式答案**：每段信息都有行内引用（[1][2][3] 格式）
- **来源卡片**：侧边栏显示完整来源列表，可展开查看
- **深度报告**：Deep Research 生成更长的综合报告
- **可视化**：支持图表、地图等可视化元素
- **分享**：可生成分享链接

### 4.7 技术架构

- **搜索增强生成（RAG 核心）**：以搜索为核心，生成为辅助
- **多模型支持**：GPT-4o、Claude、Gemini 等多模型可选
- **实时索引**：维护实时网页索引
- **知识图谱**：构建实体关系图谱辅助搜索
- **架构特点**：搜索优先（Search-first），不是推理优先（Reasoning-first）

### 4.8 定价和可用性

| 套餐 | 深度研究能力 | 价格 |
|------|------------|------|
| Free | 标准搜索 | 免费 |
| Pro Search | 有限次数 | 免费（有限） |
| Perplexity Pro（$20/月） | Pro Search + Deep Research 无限 | 付费 |
| Enterprise | 企业级 | 企业定价 |
| API | 按 token 计费 | 开发者 |

## 五、Microsoft — Copilot Deep Research / Researcher

### 5.1 产品概述

Microsoft 的深度研究能力主要体现在两个产品线：
1. **Microsoft Copilot Deep Research**：面向个人用户的独立深度研究功能（2025年推出）
2. **Microsoft 365 Copilot Researcher & Analyst**：面向企业的研究智能体（2025年3月推出）
3. **Copilot Search in Bing**：AI 驱动的新一代搜索体验（2025年4月推出）

### 5.2 产品形态和入口

- **Copilot Deep Research**：
  - 入口：Microsoft Copilot（copilot.microsoft.com）中的独立功能
  - 触发：手动选择 Deep Research 模式
- **M365 Copilot Researcher & Analyst**：
  - 入口：Microsoft 365 Copilot（企业版）
  - 自动识别研究需求或手动触发
- **Copilot Search in Bing**：
  - Bing 搜索的 AI 增强版
  - 集成在 Windows、Edge、Bing 中

### 5.3 交互方式

1. 用户输入研究主题
2. **Researcher**：自动规划研究步骤，搜索、阅读、综合
3. **Analyst**：基于研究结果进行深度分析
4. **进度展示**：实时显示研究进展
5. **Microsoft Graph 集成**：可搜索企业内部文档（M365 版）
6. 执行时间：数分钟到数小时（企业版）

### 5.4 搜索和信息收集机制

- **Bing 搜索 + OpenAI Deep Research 模型**：M365 Copilot Researcher 结合了 OpenAI 的 deep research 模型和 Microsoft 的搜索编排能力
- **企业内部搜索**：通过 Microsoft Graph 访问企业文档、邮件、Teams 消息等
- **多源整合**：外部网络 + 企业内部知识库
- **Bing 网页卡片**：Copilot Chat 中集成 Bing Web Cards 提供结构化搜索结果

### 5.5 推理/研究流程

- **双 Agent 架构**：Researcher（研究收集）+ Analyst（分析洞察）分工协作
- **编排层**：Microsoft 365 Copilot 的 advanced orchestration 能力
- **OpenAI 模型驱动**：底层使用 OpenAI 的 deep research 模型
- **日历搜索**：可搜索日历数据辅助研究

### 5.6 输出格式

- **组织化报告**：结构化的研究报告
- **引用和来源**：标注信息来源
- **关键发现**：提炼核心洞察
- **Microsoft 365 集成**：可导出到 Word、PowerPoint 等
- **Bing Copilot Search**：即时摘要式回答 + 来源建议

### 5.7 技术架构

- **混合架构**：Microsoft 搜索编排 + OpenAI 推理模型
- **Microsoft Graph**：企业数据连接层
- **Copilot Orchestrator**：多 Agent 编排框架
- **Bing AI**：底层搜索基础设施
- **关键特点**：企业级安全、合规、数据隐私

### 5.8 定价和可用性

|------|------------|------|

| Copilot Search in Bing | 免费搜索增强 | 免费 |

## 六、百度 — 文心一言深度搜索

### 6.1 产品概述

百度的深度搜索/研究能力主要通过**百度搜索的 AI 增强**和**文心一言的 Agent 能力**实现。百度作为搜索巨头，其深度研究功能与搜索引擎深度集成。

### 6.2 产品形态和入口

- **百度搜索 AI 概览**：搜索结果页面的 AI 总结和分析
