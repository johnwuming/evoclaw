# R-068 小青账APP自动记账可行性调研

> **报告日期**：2026年5月9日  
> **需求背景**：用户希望实现招商银行储蓄卡、信用卡、支付宝花呗的自动记账，记账工具为"小青账"APP，愿意通过vibe coding开发部分逻辑  
> **分析范围**：小青账APP能力边界、各数据源获取路径、自动记账技术方案、可行性评估  
> **免责声明**：本报告仅调研技术方案，涉及银行/支付宝操作需用户自行判断安全性和合规性。

## 一、小青账APP能力盘点

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| 开发商 | 有钳智能科技（郑州）有限公司 |
| 官网 | https://bill.youqian.pro |
| 帮助文档 | https://docs.bill.youqian.pro |
| 平台 | iOS / Android / 鸿蒙 |
| 核心特色 | 无广告、轻量、支持账单CSV导入、资产管理、iOS快捷指令记账 |

### 1.2 与自动记账相关的功能

| 功能 | 支持情况 | 说明 |
|------|------|------|
| **CSV账单导入** | ✅ 支持 | 核心功能，支持支付宝、微信等CSV格式批量导入 |
| **iOS快捷指令** | ✅ 支持 | 截图识别金额→跳转小青账记账，但非全自动 |
| **截图记账** | ✅ 支持 | OCR识别截图中的金额信息 |
| **资产管理** | ✅ 支持 | 可建银行卡、支付宝、微信、现金等账户 |
| **定时记账** | ✅ 支持 | 适合水电煤等周期性固定支出 |
| **开放API** | ❌ 不支持 | 小青账没有公开的开发者API或Webhook |
| **Web端** | ❌ 不支持 | 纯移动端APP，无网页版入口 |
| **自动同步银行** | ❌ 不支持 | 不像网易有钱（已关停）那样直连银行 |

### 1.3 账单导入格式

小青账支持两种导入模板：
- **模板1**：小青账自身导出格式（先导出一份作为模板，再按格式修改导入）
- **模板2**：网易有钱格式（兼容）

支持的账单来源（通过CSV）：
- 支付宝（电脑端/手机端导出的CSV）
- 微信支付
- 以及任何按模板格式整理好的CSV文件

**关键发现：小青账的自动记账上限是"半自动"——它不能主动拉取数据，但可以被动接收格式化CSV。**

## 二、三大数据源获取路径分析

### 2.1 招商银行储蓄卡

#### 获取方式

| 方式 | 自动化程度 | 格式 | 说明 |
|------|------|------|------|
| 手机APP导出流水→邮箱 | ⚠️ 半自动（需手动触发） | PDF（加密ZIP） | 招商银行APP→收支明细→右上角→导出→选时间段→输入邮箱 |
| PC网银导出 | ⚠️ 半自动 | Excel/PDF | 需登录专业版 |
| 招商银行Open API | ❌ 不可用 | — | 仅面向企业/商户，个人用户无法接入 |
| 邮件推送 | ❌ 不支持 | — | 招行储蓄卡没有定期自动推送交易明细的功能 |

#### 技术障碍

- 导出的是**加密PDF**（密码=身份证后6位），需要解析
- 已有开源项目 [beancount-cmb-importer](https://github.com/e7h4n/beancount-importer) 专门解析招行PDF流水
- 但每次导出需要**手动在APP中触发**，无法全自动

### 2.2 招商银行信用卡

#### 获取方式

| 方式 | 自动化程度 | 格式 | 说明 |
|------|------|------|------|
| 掌上生活→电邮账单 | ✅ 可自动 | 邮件（含明细） | 设置电邮对账后，**每月账单日自动推送邮件** |
| 账单补寄 | ⚠️ 半自动 | 邮件 | 手动触发补寄 |
| Open API | ❌ 不可用 | — | 同储蓄卡 |

#### 关键优势

**信用卡账单可以设置自动邮件推送！** 这是最接近"自动"的数据源：
- 在掌上生活APP中搜索"账单设置"→"电邮对账设置"
- 每月出账后自动发送含明细的邮件到指定邮箱
- 邮件中包含交易明细，可通过脚本解析

#### 技术路径

已有开源方案：[获取招商信用卡账单的方法](https://blog.heysh.xyz/2022/01/21/new-method-to-get-cmb-bill/)——解析邮件中的信用卡账单

### 2.3 支付宝（含花呗）

#### 获取方式

| 方式 | 自动化程度 | 格式 | 说明 |
|------|------|------|------|
| 支付宝APP导出 | ⚠️ 半自动 | CSV（ZIP压缩） | 我的→账单→右上角···→开具交易流水证明 |
| 电脑端导出 | ⚠️ 半自动 | CSV | alipay.com→我的账单→选时间→下载 |
| 小青账直接导入 | ✅ 已支持 | — | 小青账官方支持支付宝CSV导入 |
| 自动推送 | ❌ 不支持 | — | 支付宝没有定期推送账单的功能 |

#### 技术路径

- 支付宝CSV包含花呗消费记录（花呗是支付宝的子支付方式）
- CSV中通常有"交易分类"、"收/支"、"金额"、"交易对方"、"交易状态"等字段
- 已有大量开源Python脚本可解析支付宝CSV（见GitHub）

## 三、自动记账方案设计

### 3.1 方案概览

```
┌─────────────────────────────────────────────────┐
│                   数据获取层                      │
│                                                   │
│  招行储蓄卡          招行信用卡         支付宝/花呗    │
│  (手动触发导出)       (自动邮件推送)     (手动导出CSV)  │
│       │                  │                │       │
│       ▼                  ▼                ▼       │
│   加密PDF           邮件HTML/PDF        CSV文件      │
└───────┬──────────────────┬────────────────┬───────┘
        │                  │                │
        ▼                  ▼                ▼
┌─────────────────────────────────────────────────┐
│                 解析转换层（Python）                │
│                                                   │
│  PDF解析            邮件解析           CSV解析      │
│  (pdfplumber)       (imaplib)       (pandas)      │
│       │                  │                │       │
│       └──────────┬───────┘                │       │
│                  ▼                        │       │
│           统一小青账CSV格式                 │       │
│                  ▲                        │       │
│                  └────────────────────────┘       │
└──────────────────────┬────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│                   导入小青账                       │
│                                                   │
│  手机端：用其他应用打开CSV → 选择小青账 → 自动导入      │
│  或：直接将CSV传到手机，小青账内导入                   │
└─────────────────────────────────────────────────┘
```

### 3.2 三种自动化级别

#### Level 1：最省事（推荐起步）

**月度手动导出 + 脚本转换 + 批量导入**

| 步骤 | 操作 | 频率 | 耗时 |
|------|------|------|------|
| 1 | 招行APP导出储蓄卡流水到邮箱 | 每月1次 | 2分钟 |
| 2 | 掌上生活已自动发送信用卡账单邮件 | 自动 | 0 |
| 3 | 支付宝APP导出账单CSV | 每月1次 | 2分钟 |
| 4 | 运行Python脚本，自动解析三类账单，合并成小青账CSV | 1次点击 | 10秒 |
| 5 | CSV传到手机，小青账导入 | 每月1次 | 1分钟 |

**总耗时：每月约5分钟。需要开发：一个Python合并转换脚本。**

#### Level 2：中等自动化

**邮件自动拉取 + 自动转换**

在Level 1基础上：
- Python脚本通过IMAP自动登录邮箱，拉取招行储蓄卡流水邮件和信用卡账单邮件
- 自动解密PDF、解析内容
- 支付宝CSV仍然手动导出（无法绕开）
- 最终生成小青账格式CSV

**额外开发量：IMAP邮件拉取 + PDF解密解析模块**

#### Level 3：高度自动化（有风险）

**手机自动化 + 定时执行**

- 使用Termux（Android）或iOS快捷指令定时运行Python脚本
- 或部署在VPS上，通过cron定时执行
- 招行储蓄卡仍然需要手动在APP中触发导出（无解）
- 信用卡邮件可自动拉取
- 支付宝仍然需要手动导出（无解）

**结论：即使做到Level 3，招行储蓄卡和支付宝仍然需要每月手动触发一次导出。这是银行/支付宝的安全限制，个人用户无法绕开。**

## 四、推荐方案：Level 1 + Vibe Coding

### 4.1 开发任务清单

| # | 模块 | 技术栈 | 复杂度 | 说明 |
|---|------|--------|--------|------|
| 1 | **招行储蓄卡PDF解析** | Python + pdfplumber | ⭐⭐ | 解密ZIP→解析PDF→提取交易明细 |
| 2 | **招行信用卡邮件解析** | Python + imaplib + BeautifulSoup | ⭐⭐ | 从邮件中提取交易明细 |
| 3 | **支付宝CSV解析** | Python + pandas | ⭐ | 直接读取CSV，已有大量参考代码 |
| 4 | **统一格式转换** | Python | ⭐⭐ | 三种格式→小青账CSV模板格式 |
| 5 | **主脚本整合** | Python CLI | ⭐ | 一键执行，输出合并CSV |

### 4.2 可复用的开源项目

| 项目 | 地址 | 用途 |
|------|------|------|
| china_bean_importers | github.com/jiegec/china_bean_importers | 支持招行储蓄卡/信用卡、支付宝、微信的Beancount导入器，解析逻辑可直接复用 |
| beancount-cmb-importer | github.com/e7h4n/beancount-cmb-importer | 专门解析招行储蓄卡PDF流水 |
| KeepAccounts_v2.0 | github.com/MickLife/KeepAccounts_v2.0 | 微信+支付宝账单合并+可视化 |
| alipay-wechat-finance | github.com/zxc7563598/alipay-wechat-finance | 支付宝+微信账单解析生成财务报告 |

### 4.3 小青账CSV格式要求

根据小青账官方文档，导入模板格式为：
1. 先在小青账中导出一份账单作为模板
2. 按模板格式填充数据即可导入

**核心字段（预估）**：日期、分类、金额、账户、备注

### 4.4 Vibe Coding开发路径

```
Step 1: 先手动从小青账导出一份CSV，确认格式字段
Step 2: 写支付宝CSV→小青账CSV的转换器（最简单，验证流程）
Step 3: 写招行信用卡邮件→小青账CSV的转换器
Step 4: 写招行储蓄卡PDF→小青账CSV的转换器
Step 5: 合并成一个统一CLI工具
Step 6:（可选）部署到VPS，加邮件自动拉取
```

## 五、替代方案对比

如果你觉得"每月手动导出+脚本转换"仍然不够自动化，可以考虑以下替代：

### 5.1 换一个支持更多自动化的记账工具

| 工具 | 自动化程度 | 银行直连 | 说明 |
|------|------|------|------|
| **钱迹** | ⭐⭐⭐ | ❌ | iOS快捷指令自动记账（截图OCR），但无银行直连 |
| **iCost** | ⭐⭐⭐ | ❌ | 类似钱迹，支持CSV导入小青账/随手记等 |
| **挖财** | ⭐⭐⭐⭐ | ✅ 部分银行 | 支持微信/支付宝/银行一键导入，但需要授权 |
| **Beancount** | ⭐⭐⭐⭐⭐ | ❌ | 纯文本复式记账，配合开源importer可实现高度自动化 |
| **网易有钱** | — | — | **已关停**，曾经是最好的自动同步记账工具 |

### 5.2 纯自研方案（Beancount路线）

如果愿意投入更多开发精力：
- 使用Beancount作为记账引擎（纯文本，版本可控）
- 复用china_bean_importers的解析逻辑
- 前端用Fava（Web UI）查看
- 部署在VPS上，cron定时执行
- **但这意味着放弃小青账APP**

### 5.3 最务实的建议

**留在小青账 + 开发一个Python转换脚本（Level 1方案）**

理由：
1. 小青账APP体验好、无广告，日常随手记账够用
2. 银行和支付宝的数据获取瓶颈不在记账工具，而在数据源本身
3. 即使换成其他工具，银行流水仍然需要手动导出
4. 每月花5分钟导出+一键运行脚本，性价比最高

## 六、结论与建议

### 核心结论

1. **小青账没有API，不支持银行直连**，自动化上限是"CSV批量导入"
2. **招商银行信用卡**是最容易自动化的数据源（月度邮件自动推送）
3. **招商银行储蓄卡和支付宝**必须手动触发导出，这是安全限制，无法绕开
4. 通过Python脚本将三种数据源统一转换为小青账CSV格式，**每月操作约5分钟**，是最务实的方案
5. Vibe Coding开发量约**2-3天**（含测试），核心模块已有大量开源代码可复用

### 建议的下一步

1. **先在小青账中导出一份CSV，确认格式**（10分钟）
2. **先写支付宝→小青账转换器**（最简单，验证端到端流程）
3. 再逐步加上招行信用卡邮件解析和储蓄卡PDF解析
4. 可部署在当前VPS上（腾讯云），通过微信/邮件传CSV到手机
