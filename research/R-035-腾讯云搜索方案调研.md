# R-035 腾讯云搜索方案调研

> 调研日期：2026-04-07

## 1. 搜索引擎浏览器访问测试

| 引擎 | URL | 腾讯云可达 | 备注 |
|------|-----|-----------|------|
| **百度** | baidu.com | ✅ | 首页正常加载，但搜索触发验证码（滑块），自动化困难 |
| **Bing（国内版）** | cn.bing.com | ✅ | 完全可用，无验证码，搜索结果正常 |
| Google | google.com | ❌ | 预期不可达 |
| 搜狗 | sogou.com | 未测试 | 国内站点，预期可达 |

**结论：Bing 国内版是最可靠的浏览器搜索源，可直接通过 browser 工具执行搜索。**

## 3. 推荐方案

### 优先级排序

1. **Tavily API**（首选）
   - 国内可达，免费1000次/月，专为AI agent设计
   - 返回结构化JSON，中文搜索质量尚可
   - 接入简单：一个API key + HTTP请求
   - 官网：https://tavily.com

2. **Serper API**（备选）
   - 免费2500次/月，底层Google搜索质量高
   - 国内可达，接入同样简单
   - 中文搜索质量优于Tavily

3. **Bing Search API**（补充）
   - Azure服务，国内稳定可达
   - 中文搜索质量好
   - 免费层需绑定Azure账号

4. **Browser + Bing**（临时/兜底）
   - 已验证 cn.bing.com 完全可用
   - 通过 browser 工具 open → navigate → snapshot 实现搜索
   - 缺点：速度慢，消耗资源，非结构化输出
   - **适合作为没有API key时的临时方案**

### 临时方案（立即可用）

无需任何配置即可使用：
```
browser: navigate → https://cn.bing.com/search?q={query}
browser: snapshot → 解析搜索结果
```

### 中期方案（推荐配置）

注册 Tavily 或 Serper API key，配置为 OpenClaw 的搜索工具。两者都有充足的免费额度，覆盖日常使用。

## 4. 百度验证码问题

百度对自动化请求（尤其是从服务器IP）会触发滑块验证码。通过 browser 工具使用百度搜索不可靠，不建议作为搜索源。如果需要百度搜索，建议通过百度搜索API（需企业认证）而非浏览器。

## 5. 知识缺口

- 搜狗、360搜索未实测（优先级低，国内搜索引擎浏览器访问预期均可达）
- Brave Search API 实际可达性未验证
- 各API在腾讯云上的实际延迟未测试
