## 五、规范统一方案

### 5.1 方案 A：PATHS.md 统一路径规范

**思路**：在顶层创建 `/root/.openclaw/PATHS.md`，定义全系统所有 agent 共用的路径规范，各 AGENTS.md 仅引用不重复定义。

**优点**：
- 单点修改，全局生效
- 消除不同 AGENTS.md 中路径描述的不一致
- 新 agent 接入成本低

**缺点**：
- 需要维护额外的文档文件
- 如果 PATHS.md 与 AGENTS.md 不同步，反而造成更多混乱

**建议路径定义**：
```markdown
# OpenClaw 全局路径规范

## 系统级
- openclaw 配置: `/root/.openclaw/openclaw.json`
- 全局共享结果: `/root/.openclaw/workspace/shared/results/`
- 全局技能库: `~/.local/share/pnpm/global/5/.pnpm/openclaw@*/skills/`

## Agent 工作区
- main: `/root/.openclaw/workspace/`
- research-lead: `/root/.openclaw/workspace-research/`
- research-searcher: `/root/.openclaw/workspace-search/`
- research-reviewer: `/root/.openclaw/workspace-reviewer/`
- research-citation: `/root/.openclaw/workspace-citation/`
- dev-lead: `/root/.openclaw/workspace-dev/`
- dev-designer: `/root/.openclaw/workspace-dev-init/`
- dev-coder: `/root/.openclaw/workspace-dev-coder/`
- dev-qa: `/root/.openclaw/workspace-dev-qa/`

## 研究流程（仅研究团队）
- 研究计划: `{workspace}/research/research-plan.json`
- 知识库: `{workspace}/research/knowledge-base.json`
- 知识缺口: `{workspace}/research/gaps.json`

## 项目文件（由项目初始化脚本创建）
- 产品文档: `<项目根目录>/PRODUCT.md`
- 功能列表: `<项目根目录>/feature_list.json`
- 进度追踪: `<项目根目录>/progress.md`
- 初始化脚本: `<项目根目录>/init.sh`

## 废弃路径（请勿使用）
- ❌ `shared/projects/<项目>/PRODUCT.md` — 已废弃，使用上方项目路径
```

### 5.2 方案 B：Symlink 标准化

**思路**：对关键路径建立符号链接，保证不同 agent 使用不同路径别名时访问同一实体。

**示例**：
```bash
# 统一结果目录
/root/.openclaw/workspace/shared/results/ → /root/.openclaw/workspace-research/results/

# 研究流程目录标准化
/root/.openclaw/workspace-research/research/ → /root/.openclaw/workspace/research/  （研究主管实际使用的主目录）
```

**优点**：agent 间路径不同但指向同一实体，减少协作障碍  
**缺点**：symlink 跨越 agent workspace，可能导致权限和归属混乱；调试困难

### 5.3 方案 C：openclaw.json 标准化（推荐）

**思路**：在 `openclaw.json` 的 `agents[].params` 中增加 `paths` 字段，强制各 agent 的路径命名空间隔离，避免歧义。

```json
{
  "agents": [{
    "id": "research-lead",
    "params": {
      "paths": {
        "results": "/root/.openclaw/workspace/shared/results/",
        "researchDir": "/root/.openclaw/workspace-research/research/",
        "tmpDir": "/root/.openclaw/workspace-research/tmp/"
      }
    }
  }]
}
```

**优点**：配置即规范，路径解析在运行时完成，无需维护独立文档  
**缺点**：需要 OpenClaw 运行时支持 `params.paths` 的解析
